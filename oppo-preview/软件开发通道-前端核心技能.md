# 软件开发通道【前端】核心技能文档

## 一、核心技能框架

### 1. 专业技能

#### 1.1 程序开发

##### 开发语言
- 掌握主要开发语言的特性
- 代码性能优化
- 具备良好的编码规范和页面布局能力

##### 编码设计
- 掌握并遵循前端开发的标准规范
- 符合 web 安全设计
- 确保对接测试架构的兼容性

##### 常用框架
- 开源软件框架使用、优化
- 问题分析、扩展、选型

#### 1.2 性能调优

##### 性能优化
- 掌握页面加速、渲染等体验优化

#### 1.3 Debug能力

##### Debug能力
- 掌握主流的常用前端调试工具（Firebug、Fiddler、Chrome Dev Tools等）的使用
- 及解决问题的方法论

#### 1.4 架构设计

##### 架构设计
- 根据业务形态进行业务抽象设计
- 建设高效高质量持续交付当前与未来业务发展的方案
- 构建高性能、高安全、低流量、可扩展、可重用、易维护的 web 应用架构

### 2. 组织影响力

#### 2.1 流程制度规范的建设
- 参与建立、优化软件开发流程、制度、规范
- 并严格遵循执行

#### 2.2 经验总结与业务指导
- 将个人的工作技巧/方法、心得体会知失败的教训进行总结
- 并以咨询形式分享给他人、指导、培养新晋人员
- 促进大家学习和提高

## 四、等级标准详细说明

### 2.1 级别 - 初级工程师
**单证文档**
- 2级：web前端或对应的代码规划文档提案
- 3级：代码辅助功能文档

**开发语言**
- 功能布局

**编码设计**
- 遵循的转化、算法

**常用框架**
- 数据结构的要求选择

**单证文档**
- 代码辅助规范交流文档
- 3级：设计文档、代码

### 2.2 级别 - 中级工程师
**编码设计**
- 数据结构的优选、算法设计

**常用框架**
- 面向对象设计
- 数据结构代码运算选择

**性能调优**
- 业务模块设计文档、代码的模块化可读性、可维护、可扩展性

### 2.3 级别 - 高级工程师
**编码设计**
- 业务模块设计的设计文档、代码的模块化可读性、可维护、可扩展性

**架构设计**
- 3.1-3.2：业务模块设计的设计文档、代码的模块化可读性、可维护、可扩展性、模块划分
- 3.1为单模块、3.2为系统级的架构设计和其他架构代码和架构整体设计

### 3.1 级别 - 资深工程师
**单证文档**
- 2级：代码辅助规范交流文档
- 3级：设计文档、优化设计文档

**常用框架**
- 使用后治、注释、文档、使用场景

**架构设计**
- 选型调研、方案对比文档
- 优化设计文档

### 3.2 级别 - 技术专家
**架构设计**
- debug测试验证功能文档

### 3.3 级别 - 高级技术专家
**平台建设**
- 涉及立权改的开发及功能的方案制的代码包
- 2级：工作心得（规范目小结）
- 3级：经验总结文档、业务指导

### 4.1 级别 - 资深技术专家
**组织贡献**
- 经验总结与业务指导
- 经验总结文档与业务包的内部分享

## 五、关键文档产出要求

### 单证文档类型
1. **2级文档**：代码规范、交流协议文档
2. **3级文档**：设计文档、代码、优化设计文档

### 架构设计文档要素
- 标注、程序流、设计文案、使用场景、核心算法、怎么改的决、解决后达效果数据预测
- debug验证功能文档

### 平台建设文档
- 独立元/自动化规范代码规范制定及功能制的制定
- 涉猎成或规范、提出标准规范或事点成这
- 成或或成成、自己独定标标、欧雅提升写规范说明

## 六、能力提升路径建议

### 初级 → 中级（2.1 → 2.2）
1. 深化开发语言掌握，从基础到熟练
2. 学习面向对象编码设计
3. 掌握主流框架的使用
4. 培养基础调试能力

### 中级 → 高级（2.2 → 2.3）
1. 掌握模块化设计能力
2. 理解并应用性能优化方案
3. 深入框架原理，能够扩展和优化
4. 具备业务抽象能力

### 高级 → 资深（2.3 → 3.1）
1. 系统级架构设计能力
2. 制定技术规范和标准
3. 组件化和平台化建设
4. 团队技术影响力建设

### 资深 → 专家（3.1 → 3.2 → 3.3 → 4.1）
1. 跨系统架构设计
2. 技术方向把控和选型
3. 团队技术能力建设
4. 行业技术影响力
